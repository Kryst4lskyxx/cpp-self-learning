# Contributing

This repo is designed for steady content growth, not large one-off drops.

## Before You Add A Module

- Read [learning-path.md](learning-path.md) and [content-contract.md](content-contract.md).
- Pick one topic with a clear learner outcome.
- Keep the module small enough to finish and test in one pass.

## What To Include

- A module `README.md` with prerequisites, outcomes, and navigation.
- A short lesson and one main assignment.
- Drills that isolate one concept or failure mode at a time.
- Tests or another clear validation command.
- Expert extensions only when the core module is already useful.

## What To Avoid

- Empty placeholder folders.
- Broad refactors unrelated to the module being added.
- Long background essays that do not help the learner get started.
- Content that depends on undocumented setup or hidden tooling.

## Practical Standards

- Match the existing directory naming scheme.
- Keep commands reproducible from the repository root.
- Update the docs if you change the learner path or module contract.
- Prefer small commits that leave the repository in a usable state.
